## Qué es esto

relic.skill es un motor de inmortalidad para todo.

Puede forjar cualquier cosa que te importe —una persona, un gato, una relación, un equipo, un lugar, un instante— en un alma digital interactiva a partir de fragmentos de datos dispersos.

No es un archivo frío. Es una abuela que en Año Nuevo te pregunta si ya comiste dumplings. Es un gato que de repente empieza a hacer parkour a las tres de la mañana. Es ese product manager que siempre vuelve con otro cambio de requirements.

> El nombre viene del chip Relic de Cyberpunk 2077: un biochip capaz de almacenar una personalidad digitalizada.
> Pero este proyecto no va de cyberpunk. Va de **recordar**.

## Arquitectura del alma en cuatro dimensiones

Cada Relic captura un alma en cuatro dimensiones:

```text
        🧠 Cognición (Cognition)
        Cómo piensa, cómo decide,
        en qué cree
                |
                |
💬 Expresión ----+-------- 🎭 Conducta
Cómo habla,              Cómo actúa,
qué muletillas usa,      qué hábitos tiene,
qué tono tiene           qué patrones sigue
                |
                |
        ❤️ Emoción (Emotion)
        Qué le da alegría, qué le duele,
        cómo expresa amor, cómo gestiona conflictos
```
Cada pieza de información lleva un nivel de evidencia:

- `verbatim` — palabras exactas, sin cambiar una letra
- `artifact` — procedente de documentos, fotos o grabaciones
- `impression` — procedente de descripciones de otras personas o de recuerdos borrosos

> Las personas son inconsistentes por naturaleza. Las contradicciones no se eliminan: se etiquetan y se conservan.

## Instalación

### Opción 1: Instalar en el proyecto actual

```bash
mkdir -p .claude/skills
git clone https://github.com/Ylsssq926/relic.skill .claude/skills/relic
```

### Opción 2: Instalar en un paso con npx

```bash
npx skills add Ylsssq926/relic.skill
```

### Opción 3: Instalación global (disponible en todos los proyectos)

```bash
git clone https://github.com/Ylsssq926/relic.skill ~/.claude/skills/relic
```

### Opción 4: Otros IDE / agentes

relic.skill se basa en el estándar abierto SKILL.md y es compatible con cualquier asistente de programación con IA que lo admita:

| IDE / Agent | Método de instalación |
|-------------|-----------------------|
| **Claude Code** | `git clone` 到 `.claude/skills/relic/` |
| **Kiro** | `git clone` 到 `.kiro/skills/relic/` |
| **Cursor** | `git clone` 到 `.cursor/skills/relic/` 或项目根目录 |
| **Windsurf** | `git clone` 到 `.windsurf/skills/relic/` |
| **Cline** | `git clone` 到 `.cline/skills/relic/` |
| **OpenCode** | `git clone` 到 `.opencode/skill/relic/` |
| **Codex CLI** | `git clone` 到 `codex-skills/relic/` |
| **Augment** | `git clone` 到项目根目录 |
| **GitHub Copilot** | `git clone` 到项目根目录 |

> En principio, cualquier agent que pueda leer SKILL.md debería funcionar. Si no estás seguro, simplemente haz clone en la raíz del proyecto.

### Requisitos del entorno

- Cualquiera de los asistentes de programación con IA mencionados arriba
- Python 3.9+ (opcional, para scripts de parsing de datos)
- No necesitas GPU, modelos locales ni Docker

## Uso

### Activación por conversación (recomendado)

En Claude Code / Kiro, solo di:

```text
"Ayúdame a despertar un Relic. Quiero volver inmortal a mi abuela"
"Mi gato se fue. Quiero convertirlo en un Relic"
"Ayúdame a destilar la cultura de nuestro equipo. Estamos a punto de separarnos"
"Quiero preservar el patrón de la relación entre nosotros"
```
### Comandos Slash

```text
/relic              — Inicia el flujo de despertar de Relic
/relic-forge        — Entra directamente en el Soul Awakener
/relic-talk         — Habla con un Relic existente
/relic-shield       — Protege tu Relic
```
### Herramientas CLI

```bash
# Parsear historial de chat de WeChat
python scripts/wechat_parser.py --input ~/wechat_export/ --output data.json

# Parsear historial de chat de QQ
python scripts/qq_parser.py --input chat.txt --output data.json

# Generar un Relic
python scripts/relic_writer.py --data data.json --template human --slug grandma

# Ver si te hablaría primero
python scripts/proactive_scheduler.py --relic exes/grandma --dry-run

# Gestión de versiones
python scripts/version_manager.py snapshot --slug grandma --note "Versión 1"
python scripts/version_manager.py rollback --slug grandma --version 1
```

> Desde la v1.1.2, los Relics nuevos generados por `relic_writer.py` ya traen `proactive_config.json`, así que puedes hacer un dry-run sin escribir nada a mano y ver si hoy vendría a tocarte la puerta.

### 🐦 Integración con Feishu CLI

Integración nativa con [Feishu CLI](https://github.com/larksuite/cli) para destilación de memoria de equipo y comportamiento proactivo.

```bash
# Recolectar mensajes de grupo Feishu (últimos 30 días)
feishu-cli im list-messages --chat-id oc_xxx --start-time 2024-01-01 | \
  python scripts/feishu_forge.py --type im --output data/feishu_im.json

# Comportamiento proactivo: enviar recordatorio de reunión
python scripts/proactive_feishu.py \
  --relic exes/team-culture \
  --trigger calendar_event \
  --chat-id oc_xxx
```

| Capacidad | Comando Feishu CLI | Caso de uso |
|------|------|------|
| 💬 Mensajería instantánea | `feishu-cli im list-messages` | Destilación de cultura de equipo |
| 📄 Documentos | `feishu-cli docs export` | Destilación de expertos |
| 📊 Multidimensional | `feishu-cli base export` | Destilación de relaciones |
| 📅 Calendario | `feishu-cli calendar list-events` | Comportamiento proactivo |

🏆 **Primer lugar en el Concurso de Creadores de Feishu CLI · categoría Popularidad.** Gracias a todos los que votaron — los escenarios participantes están en la [plantilla Feishu CLI](templates/feishu-cli.md) y la [plantilla Experto](templates/expert.md).

## Declaración ética

Nos tomamos la ética en serio.

- 🔒 **Los datos no salen de tu equipo** — todos los datos del alma se almacenan localmente y no se suben a ningún servidor
- ✅ **El consentimiento va primero** — antes de destilar a otra persona, debes pasar el [protocolo de consentimiento de seis preguntas](soul-shield/consent-protocol.md)
- 🚫 **Líneas rojas claras** — no lo uses para acoso, seguimiento ni suplantación de identidad. Consulta las [líneas rojas éticas](soul-shield/ethics.md)
- 💡 **Etiquetado claro** — un Relic deja claro durante la interacción que no es una persona real
- 🧠 **Recordatorios de salud mental** — si detecta dependencia excesiva, sugerirá activamente volver al contacto social real

> Antes de destilar a tu abuela, asegúrate de que ella esté de acuerdo.
