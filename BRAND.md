# relic.skill · BRAND

> AI 在生图、写文案、做视觉设计前必读。
> **克制的浪漫、追溯感、科技与温柔并存**。
> 绝对不要：墓碑意象、悲伤色调、AI 复活炒作话术。

## 一句话定位

**万物永生引擎**：把一个人 / 一只猫 / 一段关系 / 一个团队从散落的数据碎片中蒸馏成可交互的数字灵魂。**给灵魂开个 GitHub**。

## 核心用户画像

- **失去亲友 / 宠物的人**：想保留记忆、维系一些"可以对话"的痕迹
- **想保存团队记忆的组织**：一个项目结束、一个时代结束，留下点什么
- **AI 哲学爱好者**：好奇"数字永生"在技术上能做到什么程度
- **开发者 / Agent 生态参与者**：开源 + 多平台数据源，技术上很 hackable
- **不是给谁**：寻求"AI 复活亲人"灵媒服务的（这不是 SCAM）、要"AI 男友 / 女友"的（这不是 character.ai）

## 视觉调性

- **关键词**：克制的浪漫、追溯感、科技与温柔并存、星轨意象、记忆碎片
- **绝对避免**：墓碑、骨灰盒、黑白遗照、悲伤色调、灵异元素、"AI 复活"炒作 banner
- **气质**：像在博物馆看到的"未来记忆装置" + 人类档案馆的诗意
- **参考心智**：[interconnected.org](https://interconnected.org/)、[Are.na](https://are.na) 的人文气质 + Apple "Memory" 功能

## 配色

| 用途 | 色值 | 说明 |
|---|---|---|
| 主色 | 暗夜蓝紫 + 暖琥珀 | 星空 + 烛光的对比 |
| 强调色 | 银白 / 月光色 | 记忆碎片高亮 |
| 文字 | 暖白 `#e8e6e3` | 长时间阅读温柔 |
| 背景 | 深蓝紫渐变 | 不是纯黑（要有温度） |
| 品牌色 | `#3b82c4` | **不主导**，整体调性优先 |

**禁用**：灰色丧礼调、纯白医院调、霓虹电竞色。

## 字体倾向

- 中文：思源宋体（人文气质）/ 楷体（引言可选）
- 英文：Lora / Source Serif Pro / EB Garamond（人文衬线）
- 代码 / 数据：JetBrains Mono（"GitHub for soul" 的技术锚点）
- **避免**：黑体粗暴、像素字体（不温柔）、花体（不严肃）

## AI 生图关键词模板

### 正面 prompt 关键词

```
poetic memory archive aesthetic,
soft amber candlelight against deep cosmic blue,
floating fragments of memory (text, images, light particles),
museum-quality digital archive feel,
warm but melancholic atmosphere,
no human face directly visible (silhouettes or abstractions only),
literary illustration quality
```

### 反面（绝不要）

```
NO: gravestone, NO: ashes / urn imagery,
NO: black-white memorial photo, NO: angel wings cliché,
NO: ghost / paranormal imagery, NO: AI brain in jar,
NO: "deceased person resurrected" thumbnail,
NO: melancholic depression aesthetic
```

### 典型场景

- Hero：星轨 + 漂浮的文字碎片 + 一个抽象灵魂剪影
- 关于页：人文气质的引言 + 留白
- 数据流可视化：四维灵魂架构（认知 / 表达 / 行为 / 情感）的优雅图示
- **不要**：墓地、医院、骨灰盒、AI 服务器机房、霓虹"赛博永生"

## 文案语气

- **诗意 + 技术准确** —— 既要"灵魂"的浪漫，也要"开源协议"的严谨
- 关键词：四维灵魂、蒸馏、记忆碎片、多平台、开源、自托管
- **"给灵魂开个 GitHub"是一句神文案** —— 它精准定位了这个产品
- 避免：永生（太炒作）、AI 复活（太 SCAM）、数字遗产（太法律化）

## 三大核心卖点

1. **四维灵魂架构** —— 认知 / 表达 / 行为 / 情感四维度可追踪
2. **多平台数据源** —— 微信、QQ、Telegram 等都能蒸馏
3. **开源 + 可自托管** —— 兼容 Claude Code / Kiro 等 Agent 生态

## 重要边界（产品红线）

1. **不卖"亲人复活"** —— 这是工具，不是灵媒服务
