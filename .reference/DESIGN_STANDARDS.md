# relic.skill 演示站设计规范

## 核心理念

**温度 > 科技感**  
**人性 > 功能性**  
**记住 > 炫技**

这不是一个冷冰冰的技术 Demo,而是一个有温度、能打动人的情感产品。

---

## 审美标准

### 1. 色彩使用

**主色调**: `#3b82c4` (品牌蓝)
- 传达: 信任、温暖、科技感的平衡
- 使用场景: 主按钮、链接、重点强调

**辅助色**:
- 青色 `#06b6d4`: 活力、希望
- 绿色 `#10b981`: 生命、成长
- 紫色 `#8b5cf6`: 记忆、怀念

**渐变使用原则**:
- 主渐变 (蓝-青-绿): 用于 Hero 区、重要 CTA
- 温暖渐变 (橙-粉): 用于情感化内容
- 避免过度使用渐变,保持克制

**色彩情感映射**:
- 奶奶 → 温暖橙色调
- 猫咪 → 活泼橙黄色调
- 团队 → 活力蓝绿色调
- 关系 → 柔和粉紫色调
- 地方 → 怀旧棕黄色调
- 时刻 → 电影感蓝灰色调

### 2. 字体排版

**标题**:
- 字重: 700-800 (Bold/ExtraBold)
- 行高: 1.1-1.2 (紧凑有力)
- 字号: 响应式,移动端不小于 28px

**正文**:
- 字重: 400-500 (Regular/Medium)
- 行高: 1.6-1.8 (舒适阅读)
- 字号: 16px 起步

**特殊文案**:
- 情感化文案: 适当加大字号,降低字重
- 技术说明: 使用等宽字体

### 3. 间距和留白

**原则**: 宁可多留白,不要挤

**标准间距**:
- 组件内边距: 16px/24px/32px
- 组件间距: 32px/48px/64px
- 区块间距: 80px/120px/160px

**响应式调整**:
- 移动端: 间距减半
- 平板: 间距 × 0.75
- 桌面: 标准间距

### 4. 圆角和边框

**圆角**:
- 小组件 (按钮/标签): 8px
- 卡片: 16px
- 大区块: 24px
- 头像: 50% (圆形)

**边框**:
- 默认: 1px solid rgba(255,255,255,0.1)
- 强调: 2px solid 品牌色
- 避免使用纯黑边框

### 5. 阴影和景深

**玻璃拟态效果**:
```css
background: rgba(255, 255, 255, 0.05);
backdrop-filter: blur(10px);
border: 1px solid rgba(255, 255, 255, 0.1);
box-shadow: 0 8px 32px rgba(0, 0, 0, 0.1);
```

**悬停效果**:
```css
transform: translateY(-4px);
box-shadow: 0 12px 48px rgba(59, 130, 196, 0.2);
```

**层级阴影**:
- 低: `0 2px 8px rgba(0,0,0,0.05)`
- 中: `0 8px 24px rgba(0,0,0,0.1)`
- 高: `0 16px 48px rgba(0,0,0,0.15)`

## 动效标准

### 1. 时长

- 快速交互: 150ms
- 标准过渡: 300ms
- 页面切换: 500ms
- 特殊动画: 800ms-1200ms

### 2. 缓动函数

- 入场: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo)
- 出场: `cubic-bezier(0.7, 0, 0.84, 0)` (ease-in-expo)
- 交互: `cubic-bezier(0.4, 0, 0.2, 1)` (ease-in-out)

### 3. 动画原则

- **有意义**: 每个动画都要服务于用户理解
- **不过度**: 避免炫技式动画
- **可跳过**: 重要内容不依赖动画
- **性能优先**: 使用 transform 和 opacity

## 组件设计原则

### 1. Relic 卡片

**必须传达的信息**:
- 这是谁/什么
- 什么类型 (人/宠物/关系等)
- 核心特征 (一句话)

**视觉层次**:
1. 封面图 (最吸引眼球)
2. 头像 + 名称 (识别身份)
3. 类型标签 (理解分类)
4. 简介文案 (了解特点)

**交互反馈**:
- 悬停: 轻微上浮 + 发光
- 点击: 缩放反馈
- 加载: 骨架屏,不要空白

### 2. 对话界面

**气泡设计**:
- 用户: 右对齐,蓝色渐变,圆角气泡
- Relic: 左对齐,灰色背景,带头像
- 系统: 居中,半透明,小字号

**打字机效果**:
- 速度: 50-80ms/字
- 可中断: 点击立即显示全部
- 配合光标闪烁

**情感表达**:
- 支持 emoji
- 支持语气词高亮
- 支持停顿标记 (...)

### 3. 上传区

**视觉状态**:
- 默认: 虚线边框,灰色背景
- 悬停: 边框变实,背景变亮
- 拖拽中: 高亮边框,蓝色背景
- 上传中: 进度条,百分比
- 完成: 绿色勾,文件预览

**文案**:
- 默认: "拖拽文件到这里,或点击选择"
- 拖拽中: "松开鼠标,开始上传"
- 上传中: "正在上传... 42%"
- 完成: "上传成功!"

## 文案标准

### 1. 语气

**温暖但不腻**:
- ✅ "说点什么吧,ta 在听"
- ❌ "亲爱的用户,请输入您的消息"

**真诚但不煽情**:
- ✅ "把散落的记忆碎片,锻造成可交互的数字灵魂"
- ❌ "让爱永恒,让记忆不朽,让灵魂永生"

**专业但不装**:
- ✅ "AI 会帮你提取四维特征"
- ❌ "基于深度学习的多模态特征提取引擎"

### 2. 长度

- 标题: 6-12 字
- 副标题: 15-30 字
- 按钮: 2-6 字
- 提示: 10-20 字
- 说明: 30-60 字

### 3. 禁用词汇

❌ 避免使用:
- "革命性" "颠覆性" "史无前例"
- "最好的" "最强的" "第一"
- "完美" "终极" "绝对"
- 过度的感叹号!!!

✅ 推荐使用:
- "试试看" "或许" "可能"
- "一起" "我们" "你"
- 具体的数字和事实

## 技术规范

### 1. 代码质量

**TypeScript**:
- 类型覆盖率 > 95%
- 不使用 any (除非必要)
- 接口优先于 type

**组件设计**:
- 单一职责
- Props 类型完整
- 默认值合理
- 文档注释清晰

**性能**:
- 懒加载非首屏组件
- 图片使用 Next.js Image
- 避免不必要的重渲染
- 使用 React.memo 优化

### 2. 文件组织

```
src/
├── app/              # 页面路由
├── components/       # 组件
│   ├── ui/          # 基础 UI
│   ├── relic/       # Relic 相关
│   └── layout/      # 布局
├── lib/             # 工具函数
│   ├── api.ts       # API 调用
│   ├── utils.ts     # 通用工具
│   └── constants.ts # 常量
└── styles/          # 样式
    └── globals.css  # 全局样式
```

### 3. 命名规范

**组件**: PascalCase
- `RelicCard.tsx`
- `ChatInterface.tsx`

**函数**: camelCase
- `generateCover()`
- `parseRelicData()`

**常量**: UPPER_SNAKE_CASE
- `API_BASE_URL`
- `MAX_FILE_SIZE`

**CSS 类**: kebab-case
- `relic-card`
- `chat-bubble`

### 4. Git 提交

**格式**: `<type>: <description>`

**类型**:
- `feat`: 新功能
- `fix`: 修复
- `style`: 样式调整
- `refactor`: 重构
- `docs`: 文档
- `test`: 测试

**示例**:
- `feat: add relic card component`
- `style: improve landing page spacing`
- `fix: chat bubble overflow on mobile`
