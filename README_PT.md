## O que é isto

relic.skill é um motor de imortalidade para tudo.

Ele pode despertar qualquer coisa que importe para você — uma pessoa, um pet, uma relação, uma equipe, um lugar, um momento — em uma alma digital interativa a partir de fragmentos de dados espalhados.

Não é um arquivo frio. É uma avó que no Ano-Novo Lunar pergunta primeiro se você já comeu jiaozi. É um gato que começa a fazer parkour às três da manhã. É aquele product manager que sempre aparece com mais uma mudança de requisito.

> O nome vem do chip Relic de Cyberpunk 2077 — um biochip capaz de armazenar uma personalidade digitalizada.
> Mas este projeto não é sobre cyberpunk. Ele é sobre **lembrar**.

## Arquitetura da Alma em Quatro Dimensões

Cada Relic captura uma alma em quatro dimensões:

```text
        🧠 Cognição (Cognition)
        Como pensa, como decide,
        no que acredita
                |
                |
💬 Expressão ----+---- 🎭 Comportamento
Como fala,              Como age,
quais bordões,          quais hábitos,
qual tom                quais padrões
                |
                |
        ❤️ Emoção (Emotion)
        O que traz alegria, o que dói,
        como demonstra amor, como lida com conflito
```
Cada pedaço de informação recebe um nível de evidência:

- `verbatim` — palavras exatas, sem mudar uma vírgula
- `artifact` — vindo de documentos, fotos ou gravações
- `impression` — vindo da descrição de outra pessoa ou de memória imprecisa

> As pessoas são inconsistentes por natureza. As contradições não são apagadas; são rotuladas e preservadas.

## Instalação

### Opção 1: instalar no projeto atual

```bash
mkdir -p .claude/skills
git clone https://github.com/Ylsssq926/relic.skill .claude/skills/relic
```

### Opção 2: instalar em um comando com npx

```bash
npx skills add Ylsssq926/relic.skill
```

### Opção 3: instalação global (disponível em todos os projetos)

```bash
git clone https://github.com/Ylsssq926/relic.skill ~/.claude/skills/relic
```

### Opção 4: outros IDEs / agentes

relic.skill é baseado no padrão aberto SKILL.md e é compatível com qualquer assistente de programação com IA que o suporte:

| IDE / Agent | Método de instalação |
|-------------|----------------------|
| **Claude Code** | `git clone` 到 `.claude/skills/relic/` |
| **Kiro** | `git clone` 到 `.kiro/skills/relic/` |
| **Cursor** | `git clone` 到 `.cursor/skills/relic/` 或项目根目录 |
| **Windsurf** | `git clone` 到 `.windsurf/skills/relic/` |
| **Cline** | `git clone` 到 `.cline/skills/relic/` |
| **OpenCode** | `git clone` 到 `.opencode/skill/relic/` |
| **Codex CLI** | `git clone` 到 `codex-skills/relic/` |
| **Augment** | `git clone` 到项目根目录 |
| **GitHub Copilot** | `git clone` 到项目根目录 |

> Em princípio, qualquer agent que consiga ler SKILL.md deve funcionar. Se estiver em dúvida, basta fazer clone na raiz do projeto.

### Requisitos

- Qualquer um dos assistentes de programação com IA acima
- Python 3.9+ (opcional, para scripts de parsing de dados)
- Não precisa de GPU, modelo local nem Docker

## Uso

### Acionamento por conversa (recomendado)

No Claude Code / Kiro, basta dizer:

```text
"Me ajude a despertar um Relic. Quero eternizar minha avó."
"Meu gato se foi. Quero transformá-lo em um Relic."
"Me ajude a destilar a cultura da nossa equipe. Está todo mundo prestes a seguir seu caminho."
"Quero preservar o padrão do relacionamento entre nós."
```
### Slash Commands

```text
/relic              — Inicia o fluxo de despertar de Relic
/relic-forge        — Entra direto na Soul Awakener
/relic-talk         — Conversa com um Relic já existente
/relic-shield       — Protege o seu Relic
```
### CLI Tools

```bash
# Parse do histórico de chat do WeChat
python scripts/wechat_parser.py --input ~/wechat_export/ --output data.json

# Parse do histórico de chat do QQ
python scripts/qq_parser.py --input chat.txt --output data.json

# Gera um Relic
python scripts/relic_writer.py --data data.json --template human --slug grandma

# Ver se ele puxaria assunto primeiro
python scripts/proactive_scheduler.py --relic exes/grandma --dry-run

# Version management
python scripts/version_manager.py snapshot --slug grandma --note "Primeira versão"
python scripts/version_manager.py rollback --slug grandma --version 1
```

> Desde a v1.1.2, os novos Relics gerados por `relic_writer.py` já vêm com `proactive_config.json`, então dá para fazer um dry-run sem escrever nada na mão e ver se ele bateria na sua porta hoje.

### 🐦 Integração com Feishu CLI

Use o [Feishu CLI](https://github.com/larksuite/cli) para coletar memórias de equipe do Feishu (Lark) e destilá-las em um Relic.

```bash
# Coletar mensagens de chat em grupo do Feishu
lark-im export --chat-id "oc_xxx" --output feishu_chat.json

# Coletar documentos do Feishu
lark-docs export --doc-id "doxcnxxx" --output feishu_docs.json

# Forjar um Relic a partir dos dados do Feishu
python scripts/feishu_forge.py \
  --chat feishu_chat.json \
  --docs feishu_docs.json \
  --template team-culture \
  --slug spark-studio
```

**Capacidades do Feishu CLI:**

| Módulo | O que coleta | Uso no Relic |
|--------|--------------|--------------|
| `lark-im` | Mensagens de grupo, mensagens privadas | Padrões de interação da equipe, estilo de comunicação |
| `lark-docs` | Documentos, planilhas, wikis | Conhecimento da equipe, processos de decisão |
| `lark-base` | Tabelas multidimensionais, bancos de dados | Dados estruturados da equipe, fluxos de trabalho |
| `lark-calendar` | Eventos de calendário, reuniões | Ritmo da equipe, momentos importantes |

🏆 **1º lugar no Concurso de Criadores Feishu CLI — categoria Popularidade.** Obrigado a todos que votaram — veja o [modelo Feishu CLI](templates/feishu-cli.md) e o [modelo Especialista](templates/expert.md).

## Declaração Ética

Levamos ética a sério.

- 🔒 **Os dados ficam locais** — Todos os dados da alma ficam armazenados na sua máquina e nunca são enviados para nenhum servidor
- ✅ **Consentimento primeiro** — Antes de destilar outra pessoa, você precisa passar pelo [protocolo de consentimento em seis perguntas](soul-shield/consent-protocol.md)
- 🚫 **Red lines claras** — Não use isto para assédio, perseguição ou impersonation. Veja as [red lines éticas](soul-shield/ethics.md)
- 💡 **Identificação explícita** — Um Relic deixa claro durante a interação que não é uma pessoa real
- 🧠 **Lembretes de saúde mental** — Se houver sinais de dependência excessiva, ele sugere voltar ao convívio social do mundo real

> Antes de destilar a sua avó, confirme se ela topa ser destilada.
