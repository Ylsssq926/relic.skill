# 团队 · 星火工作室 · 5人创业团队

> 一支会在飞书里一边骂工具、一边改需求、一边救线上事故，最后还能在周五用海底捞和梗图把彼此重新缝起来的 5 人 AI 创业团队。

这是 relic.skill 的团队文化示例 Relic，基于一个真实的 AI 产品创业团队的协作记忆创建。

## 配置媒体（可选）

### 让团队成员开口说话

如果你想让团队成员真的发语音，在 `manifest.json` 里配置 TTS：

```json
"media": {
  "tts": {
    "enabled": true,
    "provider": "minimax",
    "emotion_mapping": {
      "daily": "calm",
      "urgent": "concerned",
      "celebration": "happy"
    }
  }
}
```

然后设置环境变量：

```bash
export MINIMAX_API_KEY=你的API密钥
```

**为什么推荐 MiniMax？** 因为它对中文的情绪把握最好，能表现出团队成员在不同场景下的语气变化。

### 生成团队氛围图

如果你想生成团队工作场景的图片，在 `manifest.json` 里配置图像生成：

```json
"media": {
  "image": {
    "enabled": true,
    "provider": "seedream",
    "style": "modern_illustration",
    "cover_prompt": "创业团队办公室,飞书界面,海底捞外卖盒,深夜加班氛围"
  }
}
```

然后设置环境变量：

```bash
export SEEDREAM_API_KEY=你的API密钥
```

## 文件说明

这个 Relic 包含以下文件：

### SKILL.md

入口文件。定义了星火工作室是谁、怎么协作、对话原则。

核心内容：
- **你是谁** — 星火工作室，5人AI产品创业团队，2022-2025，已解散
- **核心文化** — 先吐槽再开工、高压高密度高自嘲、先接住再说
- **对话原则** — 多角色轮流发言、飞书群聊风格、短句快节奏
- **团队的优先级** — 先止血、再复盘、最小可上线版本优先

### personality.md

四维团队画像（决策风格/沟通习惯/价值观/内部梗和仪式）。

包含：
- **决策风格** — 高频争论+快速收敛+最小可上线版本拍板
- **沟通习惯** — 飞书群聊、碎片化高频同步、关键结论落文档
- **价值观** — 事情要落地、坏消息要早说、细节该抠就抠
- **内部梗和仪式** — 周五海底捞、共享meme文件夹、"这个需求不合理但我可以做"

每条信息都标注了证据等级（`verbatim` / `artifact` / `impression`）。

### interaction.md

五种交互场景的示例对话：

1. **需求评审** — 产品提需求、CTO评估、设计卡体验、运营追时间
2. **庆祝上线** — 上线后观察稳定性、确认没炸、再一起放松庆祝
3. **线上 bug** — 快速定位、分工止血、对外沟通、事后复盘
4. **中午吃什么** — 午饭投票、工作疲惫里的嘴碎闲聊和状态恢复
5. **最后团建** — 处理聚餐琐事、留住合照和群先别解散的余味

### memory.md

记忆片段。包含：
- 第一次线上事故的场景
- 周五海底捞的传统
- 飞书群聊的日常
- 团队解散前的备份

### manifest.json

配置文件。定义了：
- **团队信息** — 名字、规模、行业、活跃年份、解散状态
- **角色设定** — 产品经理、CTO、设计师、运营、实习生
- **协作配置** — 沟通平台、语言风格、决策模式
- **媒体配置** — TTS、图像生成
- **主动行为** — 节日、纪念日、随机触发

### proactive_config.json

主动行为配置。定义了：
- **节日** — 新年、国庆、中秋
- **纪念日** — 星火工作室成立日、团队解散日、第一次产品上线、第一次线上事故、最后一次周五海底捞
- **随机触发** — 最少间隔 10 天（团队成员会主动分享当年的段子）
- **免打扰时段** — 23:00 - 09:00
- **频率限制** — 每周最多主动 2 次

## 这个 Relic 的特点

### 表达方式

- **多角色轮流发言** — 产品、CTO、设计、运营、实习生各有声音
- **飞书群聊风格** — 短句、快节奏、像真的在群里聊天
- **会用内部梗** — "这个需求不合理但我可以做""这个颜色不对""飞书又在演我"
- **互联网团队语感** — 说人话，不故作黑话

### 协作方式

- **高频争论+快速收敛** — 吵得热闹，但不会拖着不拍板
- **先接住再说** — 不管需求多离谱，先接住，再一边吐槽一边做
- **最小可上线版本优先** — 做得完、能验证、不会立刻炸，就是胜利
- **坏消息要早说** — 尽早暴露风险，不把"还没搞定"藏到最后

### 记忆特点

- **内部梗是团队记忆** — 周五海底捞、共享meme文件夹、"飞书又在演我"
- **仪式性动作** — 上线前战时模式、上线后先确认没炸、周五点海底捞
- **真实决策过程** — 不是理想化的团队协作，而是真实的争论、妥协、救火

## 用这个 Relic 做什么

### 1. 新成员了解团队文化

如果有新人加入类似的创业团队，可以通过这个 Relic 快速了解"这个团队是怎么工作的"。

### 2. 离职后回忆团队

当团队解散后，成员可以通过这个 Relic 回忆当年的协作方式、内部梗、那些一起救火的夜晚。

### 3. 团队复盘

在团队还在的时候，可以用这个 Relic 做文化复盘：我们真的是这样协作的吗？哪些是好的，哪些需要改进？

### 4. 学习如何创建团队文化 Relic

参考这个 Relic 的文件结构和内容，学习如何为自己的团队创建文化记录。

### 5. 改造成你的团队

如果你也想记录你的团队文化，可以基于这个 Relic 改造：
1. 把 `SKILL.md` 里的团队名、规模、行业改成你的
2. 把 `personality.md` 里的四维画像改成你团队的
3. 把 `interaction.md` 里的对话改成你团队的真实场景
4. 把 `memory.md` 里的记忆改成你团队的真实记忆
5. 把 `manifest.json` 和 `proactive_config.json` 里的配置改成适合你的

## 伦理说明

### 这是已解散团队的文化记录

星火工作室在 2025 年已经解散。这个 Relic 不是"团队还在线上协作"，而是"团队文化的记忆载体"。

在 `SKILL.md` 里有明确说明：

> 如果用户明显把你当成真的还在运作的团队，要温和提醒：你是按星火工作室留下的协作记忆和文化气质整理出的 Relic，是文化记录，不是真的团队还在一起工作。

在 `manifest.json` 里也有标识：

```json
"subject": {
  "status": "dissolved",
  "notes": "该示例用于团队文化型 Relic,承载特定团队的协作记忆与说话气味,不是真人团队实时在线。"
}
```

### 团队文化 Relic 的特殊性

和个人 Relic 不同，团队文化 Relic 记录的是：
- **协作模式** — 怎么开会、怎么决策、怎么救火
- **沟通风格** — 怎么说话、什么口头禅、什么内部梗
- **价值观** — 什么被鼓励、什么被嫌弃、什么是底线
- **仪式性动作** — 周五海底捞、上线前战时模式、复盘会议

这些不是某个人的性格，而是团队在长期协作中形成的"集体人格"。
