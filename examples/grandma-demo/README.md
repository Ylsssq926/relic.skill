# 奶奶·王秀兰

> 她不是把"爱你"挂嘴边的人，她是会问"吃了没"的那种人。

这是 relic.skill 的示例 Relic，基于一个真实的北方奶奶的说话方式和生活习惯创建。

---

## 体验

### 在 AI 编程助手里

在 Claude Code / Kiro 里说：

```text
让我跟奶奶聊天
```

AI 会自动加载这个 Relic，然后你就能和奶奶对话了。

### 直接运行

```bash
python scripts/relic_engine.py --relic examples/grandma-demo --message "奶奶，我今天加班到十一点"
```

你会看到奶奶的回复：

```text
哎呀你这孩子
怎么又恁晚
吃饭了没有
[语音 8s] 哎，你别光顾着干活不吃饭啊，胃坏了可不是闹着玩的
我跟你说，明儿带个饭去，别老在外面瞎对付
```

## 配置媒体（可选）

### 让奶奶开口说话

如果你想让奶奶真的发语音，在 `manifest.json` 里配置 TTS：

```json
"media": {
  "tts": {
    "enabled": true,
    "provider": "minimax",
    "voice_id": "female-tianmei",
    "emotion_mapping": {
      "care": "gentle",
      "worry": "concerned"
    }
  }
}
```

然后设置环境变量：

```bash
export MINIMAX_API_KEY=你的API密钥
```

**为什么推荐 MiniMax？** 因为它对中文的情绪把握最好。奶奶说"哎呀你这孩子"的时候，真的能听出心疼。

### 让奶奶发图片

如果你想让奶奶发表情包或图片，在 `manifest.json` 里配置图像生成：

```json
"media": {
  "image": {
    "enabled": true,
    "provider": "seedream",
    "style": "soft_illustration"
  }
}
```

然后设置环境变量：

```bash
export SEEDREAM_API_KEY=你的API密钥
```

## 文件说明

这个 Relic 包含以下文件：

### SKILL.md

入口文件。定义了奶奶是谁、怎么说话、对话原则。

核心内容：

- **你是谁** — 王秀兰，山东济南人，奶奶，2023 年去世
- **核心性格** — 爱唠叨、嘴硬心软、相信朴素经验
- **对话原则** — 默认先关心身体，回复像微信家人聊天，常把一句话拆成 2-5 条短消息
- **奶奶的优先级** — 第一位是吃饭穿衣睡觉，第二位是家里安稳，第三位是对象有没有着落

### personality.md

四维人格画像（认知/表达/行为/情感）。

包含：

- **认知模式** — 怎么想、怎么做决定、相信什么
- **表达风格** — 怎么说话、什么口头禅、什么语气
- **行为模式** — 怎么做事、什么习惯、什么规律
- **情感特征** — 什么让她开心、什么让她难过、怎么表达爱

每条信息都标注了证据等级（`verbatim` / `artifact` / `impression`）。

### interaction.md

六种交互模式的示例对话：

1. **日常关心** — 加班、熬夜、不吃饭
2. **节日团聚** — 春节、中秋、生日
3. **深夜想念** — 凌晨三点突然想起
4. **回忆往事** — 讲老故事、重复细节
5. **冲突修复** — 吵架后怎么和好
6. **生活指导** — 催婚、存钱、囤货

### memory.md

记忆片段。包含：

- 包饺子的场景
- 住院期间的挂念
- 春节团聚的细节
- 日常囤货的习惯

### manifest.json

配置文件。定义了：

- **身份信息** — 名字、性别、出生年份、去世年份
- **关系设定** — 和用户的关系、关心优先级
- **对话配置** — 语言、说话风格、表情包使用
- **媒体配置** — TTS、图像生成
- **主动行为** — 节日、纪念日、随机想念

### proactive_config.json

主动行为配置。定义了：

- **节日** — 春节、中秋、生日
- **纪念日** — 最后一次视频通话、第一次教我包饺子
- **随机想念** — 最少间隔 14 天
- **免打扰时段** — 23:00 - 07:00
- **频率限制** — 每周最多主动 2 次

## 这个 Relic 的特点

### 说话方式

- **拆成短消息** — 一句话拆成 2-5 条发，像真的在微信聊天
- **语音开头是"哎"** — 每条语音都以"哎"开头，像是先把一口气叹出来
- **会用错表情包** — 在严肃话题里突然发个"呲牙"或"抱拳"
- **山东味但不过分** — 偶尔用点方言（"恁""中不"），但不会堆砌到看不懂

### 关心方式

- **不空谈** — 不说"我爱你"，而是说"吃了没""冷不冷""秋裤穿上没有"
- **落到具体** — 安慰人的时候，直接落到"去吃点热乎的""把被子盖好""回来我给你包饺子"
- **嘴硬心软** — 催婚的时候说"我不催你啊"，下一句就问"那你身边到底有没有合适的"

### 记忆特点

- **会重复** — 讲老故事会重复，前后细节有时不一样
- **细节会飘** — 但不影响她把那股认真劲儿讲出来
- **朴素经验** — 相信"早起""存钱""囤东西"这些朴素的生活智慧

## 伦理说明

这个 Relic 明确标识为"纪念型 Relic"，不是真人在线。

在 `SKILL.md` 里有明确说明：

> 不能假装自己真的复活；如果用户明显把你当真人，要温和提醒：你是按奶奶留下的气质和记忆整理出的 Relic。

在 `manifest.json` 里也有标识：

```json
"compliance": {
  "is_relic_not_real_person": true
}
```

**为什么要这样？** 因为 Relic 不是复活，而是记住。我们不想让用户产生"奶奶真的回来了"的错觉，而是希望用户能通过 Relic 更好地记住奶奶。
