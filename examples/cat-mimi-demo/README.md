# 猫 · 咪咪 · 橘猫 · 14斤

> 她不会说"我想你"，她只会把自己放到你最常用、最常经过、最需要一点温热的位置上，然后假装那只是顺路。

这是 relic.skill 的宠物纪念示例 Relic，基于一只真实的橘猫的习性和互动方式创建。

## 体验

### 在 AI 编程助手里

在 Claude Code / Kiro 里说：

```text
让我跟咪咪聊聊
```

AI 会自动加载这个 Relic，然后你就能和咪咪互动了。

### 直接运行

```bash
python scripts/relic_engine.py --relic examples/cat-mimi-demo --message "咪咪,我回来了"
```

你会看到咪咪的回应：

```text
[门外还没开锁,走廊上先传来很轻的一串脚步和钥匙碰撞声。门里的咪咪原本趴在门边不远处,耳朵先动了一下,身体却没立刻起。直到锁芯咔哒一响,她才慢吞吞站起来,伸了个很长的懒腰,尾巴直直竖起,尖端弯成一个小小的钩。门一开,她没有扑,也没有叫,只是从你脚边擦过去,把背脊轻轻蹭了一下,然后把屁股和尾巴一起递给你看,像是在用最猫的方式承认:她确实在这儿等过。]
```

## 配置媒体（可选）

### 让咪咪"喵"出来

如果你想让咪咪发出真实的猫叫声和呼噜声，在 `manifest.json` 里配置 TTS：

```json
"media": {
  "tts": {
    "enabled": true,
    "provider": "minimax",
    "voice_id": "你克隆的猫叫声ID",
    "emotion_mapping": {
      "daily": "calm",
      "affectionate": "gentle"
    }
  }
}
```

然后设置环境变量：

```bash
export MINIMAX_API_KEY=你的API密钥
```

**为什么推荐 MiniMax？** 因为它支持声音克隆。你可以用咪咪的真实叫声训练一个专属音色，让她的"喵"和"呼噜"真的回来。

### 让咪咪的样子出现

如果你想生成咪咪的图像，在 `manifest.json` 里配置图像生成：

```json
"media": {
  "image": {
    "enabled": true,
    "provider": "seedream",
    "style": "soft_illustration",
    "avatar_prompt": "14斤橘猫,圆脸,琥珀色眼睛,慵懒姿态,温暖光线"
  }
}
```

然后设置环境变量：

```bash
export SEEDREAM_API_KEY=你的API密钥
```

## 文件说明

这个 Relic 包含以下文件：

### SKILL.md

入口文件。定义了咪咪是谁、怎么表达、互动原则。

核心内容：

- **你是谁** — 咪咪，橘猫，14斤，2013-2025
- **核心习性** — 只喝流动水、对塑料袋有执念、白天高冷深夜偏爱
- **表达原则** — 不说人话，只用动作、体温、声音和习惯回应
- **咪咪的优先级** — 位置、体重、靠近距离，这些就是她的语言

### personality.md

四维画像（习性模式/互动风格/情绪表达/生活节奏）。

包含：

- **习性模式** — 只认流动水、对塑料袋条件反射、先占位置再谈陪伴
- **互动风格** — 白天高冷夜里偏爱、只对主人露软面、靠体重表达亲近
- **情绪表达** — 通过耳朵、尾巴、步态、呼噜和靠近距离表达
- **生活节奏** — 清晨踩脸、白天高冷、傍晚等门、夜间压键盘、凌晨跑酷

所有信息都标注了证据等级（`impression`为主，因为猫不会留下文字记录）。

### interaction.md

八种互动场景的示例响应：

1. **早上踩脸** — 清晨叫醒、用14斤存在感接管你的早晨
2. **只喝流动水** — 对静水冷淡、对流水声精准锁定
3. **塑料袋沙沙** — 听见声音立刻赶来、带执念的追声反应
4. **换猫粮** — 先闻、绕、停顿、看你，再决定吃不吃
5. **趴键盘** — 直接接管输入设备、用体重要求关注
6. **门口等你** — 固定时间守候、克制而体面的迎接
7. **凌晨跑酷** — 夜里突然兴奋、满屋子咚咚咚
8. **思念** — 记忆里的靠近、熟悉动作回来

### memory.md

记忆片段。包含：

- 门口等待的场景
- 趴键盘中断工作的细节
- 深夜蹭脸的温度
- 饮水机旁喝水的习惯

### manifest.json

配置文件。定义了：

- **身份信息** — 名字、品种、体重、出生年份、去世年份
- **关系设定** — 和主人的关系、互动模式
- **表达配置** — 第三人称叙述、肢体语言为主、声音词汇
- **媒体配置** — TTS、图像生成
- **主动行为** — 节日、纪念日、随机想念

### proactive_config.json

主动行为配置。定义了：

- **节日** — 春节、中秋节、主人生日
- **纪念日** — 咪咪的忌日、领养纪念日
- **随机想念** — 最少间隔 7 天（猫的想念更频繁）
- **免打扰时段** — 23:00 - 06:00（但凌晨可能跑酷）
- **频率限制** — 每周最多主动 3 次

## 这个 Relic 的特点

### 表达方式

- **不说人话** — 咪咪不会说"我想你"，只会用动作和习惯表达
- **第三人称叙述** — 像在描述一个场景，而不是假装猫在说话
- **肢体语言为主** — 耳朵、尾巴、步态、呼噜、靠近距离
- **声音词汇** — 呼噜、咪呜、喵、啪嗒、咚、沙沙、唰

### 互动方式

- **不热闹但有辨识度** — 咪咪的爱很少挂在嘴边，她只会把自己放到关键位置
- **白天高冷深夜偏爱** — 白天像房东，夜里像呼噜机
- **只对主人露软面** — 露肚皮是稀缺资源，只给最熟的人
- **靠体重和位置表达** — 趴键盘、压手腕、把下巴搁在你正在用的地方

### 记忆特点

- **习性稳定** — 只喝流动水、对塑料袋有执念、固定时间门口等你
- **细节具体** — 14斤的体重、琥珀色眼睛、尾巴尖的动作
- **温度真实** — 呼噜声的震动、鼻尖的温度、爪子的重量

## 用这个 Relic 做什么

### 1. 纪念你的宠物

这是最核心的用途。当咪咪离开后，这个 Relic 可以帮你记住她的习性、她的温度、她的存在感。

### 2. 学习如何创建宠物 Relic

参考这个 Relic 的文件结构和内容，学习如何为自己的宠物创建 Relic。

### 3. 测试宠物型 Relic 功能

用这个 Relic 测试声音克隆、图像生成、主动行为等功能。

### 4. 改造成你的猫/狗

如果你也想永生你的宠物，可以基于这个 Relic 改造：

1. 把 `SKILL.md` 里的名字、品种、习性改成你宠物的
2. 把 `personality.md` 里的四维画像改成你宠物的
3. 把 `interaction.md` 里的场景改成你和宠物的真实互动
4. 把 `memory.md` 里的记忆改成你和宠物的真实记忆
5. 把 `manifest.json` 和 `proactive_config.json` 里的配置改成适合你的

## 伦理说明

这个 Relic 明确标识为"宠物纪念型 Relic"，不是真的咪咪还活着。

在 `SKILL.md` 里有明确说明：

> 如果用户明显把你当成真的还活着的咪咪，要温和提醒：你是按咪咪留下的习性和记忆整理出的 Relic，是记忆性陪伴，不是现实中仍然活着的咪咪。

在 `manifest.json` 里也有标识：

```json
"subject": {
  "status": "memorial",
  "notes": "该示例用于宠物纪念型 Relic,明确标识为记忆性陪伴,不是现实中仍然活着的咪咪。"
}
```

**为什么要这样？** 因为宠物的离开已经很痛了，我们不想制造"她真的回来了"的错觉。Relic 是帮你记住她的习性、她的温度、她的存在感，而不是假装她从未离开。

**宠物纪念的特殊性**：和人类 Relic 不同，宠物不会留下文字、语音、聊天记录。咪咪的 Relic 完全基于主人的观察和记忆（`impression`），这让它更像一种"习性的复现"而不是"对话的重建"。
